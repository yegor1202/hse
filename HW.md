2. Постановка задачи

Необходимо реализовать код на языке **C++ (C++11 и
выше)**, который обеспечивает обработку задач несколькими потоками. Для этого необходимо:

1.  Реализовать потокобезопасную очередь задач TaskQueue .
2.  Создать несколько рабочих потоков worker.
3.  Потоки получают задания из очереди и обрабатывают из 1 секунду.
4.  Использовать механизмы синхронизации стандартной библиотеки C++.


3. Реализация программы:
    3.1 Заголовочные файлы:
        #include <iostream> - ввод-вывод
        #include <thread> - Создание потоков
        #include <vector> - работа с дин. массивом потоков
        #include <queue> - дял создания очереди
        #include <mutex> - для работы с мьютексом
        #include <condition_variable> - для работы с condition_variable
        #include <chrono> - для таймера обработки
    3.2 Класс TaskQueue:
    Хранит очередь, мьютекс для очереди и condition_variable для контроля доступа потоков к данным и предовращения гонок данных.

    Метод push - принимает целое число - номер задачи, блокирует мьютекс, добавляет номер задачи и пробуждает один из ждущих потоков.
    Как только произойдет выход из этого куска кода, удалится старый Lock и мьютекс освободится (ввиду использования std::unique_lock<std::mutex>)

    Метод pop - принимает целое число - номер задачи, блокирует мьютекс, если очередь пуста, мьютекс освобождается и поток засыпает. Иначе удаляет номер процесса и аналогично Push освобождает мьютекс.
    std::mutex coutMutex; - мьютекс для корректного вывода.

    3.3 Функция worker:
    Ждет задачу с помощью цикла и pop (Это НЕ busy-wait, т.к в случае отсутствия задач поток засыпает, а не постоянно крутит условие, строчка cv.wait... в pop() это обеспечивает)
    если на вход получает -1 - поток завершается (задач нет), иначе ждет 1 секунду и выводит сообщение об обработанном процессе.

    3.4 Основная функция main:
    Создает очередь queue типа TaskQueue, создает динамический массив работников (потоков), их число N = 4 (например) по следущей схеме:
    В очередь потоков добавляется обьект с находящейся в нем функцией, ссылкой на очередь (для этого используется ref), и номером.
    затем очередь наполняется 20-ю процессами и 4-мя "-1" для каждого потока, чтобы в конце завершиться.

    В конце ждем завершения всех потоков с поомщью join для каждого потока
